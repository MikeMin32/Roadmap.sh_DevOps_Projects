# GitHub Pages Deployment

## Overview
Automate the deployment of a static website to GitHub Pages using GitHub Actions.

## Skill Level
Beginner

## Domain
CI/CD

## Objectives
- Learn GitHub Actions basics
- Automate deployment workflows
- Understand CI/CD concepts
- Work with static site generators

## Project Requirements
Set up automated deployment:
- Create a static website (HTML or using Jekyll/Hugo)
- Configure GitHub Pages in repository settings
- Create GitHub Actions workflow
- Automate build and deployment on push to main branch
- Add status badges to README
- Implement basic testing (link checking, HTML validation)

## Suggested Technologies
- GitHub Actions
- GitHub Pages
- Jekyll, Hugo, or plain HTML/CSS/JS
- Markdown

## Learning Outcomes
- GitHub Actions workflow syntax
- YAML configuration
- CI/CD pipeline concepts
- Static site deployment

## Next Steps
After completing this project, try:
- Adding custom domain with HTTPS
- Implementing branch-based preview deployments
- Adding automated testing (lighthouse, accessibility)
- Integrating with a CMS
- Setting up redirects and custom 404 pages
