# Nginx Log Analyser

## Overview
Build a tool to parse and analyze Nginx access logs to extract useful insights.

## Skill Level
Beginner

## Domain
CLI / Log Analysis

## Objectives
- Parse structured log files
- Extract and aggregate data
- Generate statistics and reports
- Visualize data in terminal

## Project Requirements
Create a script that analyzes Nginx logs and displays:
- Total number of requests
- Requests per status code (200, 404, 500, etc.)
- Top 10 IP addresses by request count
- Top 10 requested URLs
- Traffic by hour/day
- Average response size

## Suggested Technologies
- Python/Bash/Go
- Regular expressions
- Text processing tools: `awk`, `sed`, `grep`

## Learning Outcomes
- Log parsing and pattern matching
- Data aggregation
- Statistical analysis
- Report generation

## Next Steps
After completing this project, try:
- Adding support for custom log formats
- Detecting suspicious patterns (potential attacks)
- Creating HTML/CSV reports
- Real-time log monitoring mode
